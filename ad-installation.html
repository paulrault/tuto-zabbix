Chapitre 11 Active Directory GUI

Installation AD DS

Installation du rôle Active Directory Domain Services sur Windows Server 2025 et promotion du serveur en contrôleur de domaine.

01

Introduction

Active Directory Domain Services (AD DS) est le service d'annuaire de Microsoft. Il permet de centraliser la gestion des utilisateurs, des ordinateurs, des groupes et des stratégies de sécurité au sein d'un domaine Windows.

ConceptDescription
DomaineEnsemble d'objets (users, PCs, groupes) partageant une base de données AD commune
Contrôleur de domaine (DC)Serveur qui héberge et gère la base de données AD DS
ForêtEnsemble de domaines partageant un schéma AD commun
Unité d'organisation (OU)Conteneur permettant d'organiser les objets AD et d'appliquer des GPO
GPOStratégie de groupe appliquée aux utilisateurs et ordinateurs d'une OU
ℹ️

Dans ce tutoriel, le contrôleur de domaine est la VM Windows Server 2025 avec l'IP 192.168.2.16. Le domaine créé sera DOMAINE.PAULR.

02

Ajout du rôle AD DS

Ouvrir le Gestionnaire de serveur (Server Manager) qui se lance automatiquement au démarrage.

Gestionnaire de serveur→Gérer→Ajouter des rôles et fonctionnalités
  1. Cliquer sur Suivant sur la page d'introduction
  2. Type d'installation → "Installation basée sur un rôle ou une fonctionnalité" → Suivant
  3. Sélection du serveur → choisir le serveur local → Suivant
  4. Dans la liste des rôles, cocher "Services de domaine Active Directory"
  5. Une fenêtre propose d'ajouter les fonctionnalités requises → cliquer "Ajouter des fonctionnalités"
  6. Cliquer Suivant jusqu'à la page de confirmation
  7. Cliquer "Installer" et attendre la fin de l'installation (2-3 minutes)
⚠️

Ne pas fermer le Gestionnaire de serveur pendant l'installation. Attendre que la barre de progression atteigne 100%.

03

Promotion en contrôleur de domaine

Une fois le rôle installé, une notification apparaît dans le Gestionnaire de serveur (drapeau jaune en haut à droite) :

Gestionnaire de serveur→⚑ Notifications→Promouvoir ce serveur en contrôleur de domaine

L'assistant de configuration AD DS s'ouvre.

04

Configuration de la nouvelle forêt

Suivre l'assistant de configuration avec les paramètres suivants :

Étape 1 — Configuration du déploiement :

ChampValeur
Opération de déploiementAjouter une nouvelle forêt
Nom de domaine racineDOMAINE.PAULR

Étape 2 — Options du contrôleur de domaine :

ChampValeur
Niveau fonctionnel de la forêtWindows Server 2016 (ou supérieur)
Niveau fonctionnel du domaineWindows Server 2016 (ou supérieur)
Serveur DNS✅ Coché
Catalogue global (GC)✅ Coché
Mot de passe DSRMDéfinir un mot de passe sécurisé (récupération d'urgence)

Étape 3 — Options supplémentaires :

ChampValeur
Nom NetBIOS du domaineDOMAINE (généré automatiquement)

Cliquer Suivant jusqu'à la page de vérification des prérequis. Des avertissements peuvent apparaître (normaux). Cliquer sur "Installer".

⚠️

Le serveur redémarre automatiquement après la promotion. C'est normal. La session de connexion passera de Administrateur à DOMAINEAdministrateur.

05

Vérification

Après redémarrage, vérifier que le domaine est opérationnel :

Gestionnaire de serveur→Outils→Utilisateurs et ordinateurs Active Directory

La console ADUC (Active Directory Users and Computers) doit s'ouvrir et afficher l'arborescence du domaine DOMAINE.PAULR avec les conteneurs par défaut :

ConteneurDescription
BuiltinGroupes locaux intégrés (Administrateurs, Utilisateurs...)
ComputersOrdinateurs rejoignant le domaine (par défaut)
Domain ControllersContrôleurs de domaine
UsersUtilisateurs et groupes par défaut
✅

Le contrôleur de domaine DOMAINE.PAULR est opérationnel. Vous pouvez maintenant créer des utilisateurs, des unités d'organisation et des GPO.
