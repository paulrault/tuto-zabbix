Chapitre 19 Proxy pfSense

Proxy Squid

Installation et configuration du proxy Squid intégré dans pfSense pour le cache web, le filtrage de contenu via SquidGuard et la journalisation du trafic.

01

Introduction

Squid est un proxy web open source qui s'intercale entre les clients du réseau local et Internet. Intégré à pfSense via un paquet, il offre plusieurs fonctionnalités :

FonctionnalitéDescription
Cache webStocke localement les pages visitées pour accélérer les accès répétés et réduire la bande passante
Proxy transparentIntercepte automatiquement le trafic HTTP sans configuration sur les postes clients
JournalisationEnregistre toutes les requêtes web des utilisateurs
Filtrage (SquidGuard)Bloque l'accès à des catégories de sites (adultes, réseaux sociaux, jeux...)
⚠️

Squid intercepte uniquement le trafic HTTP (port 80) en mode transparent. Le trafic HTTPS (port 443) nécessite une configuration SSL Bump plus complexe non couverte ici.

02

Installation du paquet Squid

Dans l'interface web de pfSense :

System→Package Manager→Available Packages
  1. Rechercher "squid" dans la barre de recherche
  2. Cliquer sur Install à côté de squid
  3. Confirmer en cliquant Confirm
  4. Attendre la fin de l'installation (1-2 minutes)

Installer également SquidGuard pour le filtrage de contenu :

  1. Rechercher "squidguard" dans Available Packages
  2. Cliquer Install → Confirm
✅

Après installation, les menus Services → Squid Proxy Server et Services → SquidGuard Proxy Filter apparaissent dans la navigation.

03

Configuration générale

Services→Squid Proxy Server→General
ParamètreValeurDescription
Enable Squid Proxy✅ CochéActive le proxy
Listen InterfaceLANInterface sur laquelle Squid écoute
Proxy Port3128Port d'écoute du proxy
Allow Users on Interface✅ CochéAutorise les clients LAN à utiliser le proxy
Transparent HTTP Proxy✅ CochéMode transparent — pas de config sur les clients
Transparent Proxy InterfaceLANInterface pour l'interception transparente
Enable Access Logging✅ CochéJournalise toutes les requêtes
Log Store Directory/var/squid/logsRépertoire des logs

Cliquer Save en bas de page.

ℹ️

En mode transparent, pfSense ajoute automatiquement une règle NAT pour rediriger le trafic HTTP du port 80 vers le port 3128 de Squid. Aucune configuration n'est requise sur les postes clients.

04

Configuration du cache

Services→Squid Proxy Server→Local Cache
ParamètreValeur recommandée
Cache Replacement PolicyHeap LFUDA (meilleur taux de succès)
Hard Disk Cache Size1000 Mo (adapter selon l'espace disponible)
Hard Disk Cache Location/var/squid/cache
Minimum Object Size0 Ko
Maximum Object Size512 Mo
Memory Cache Size256 Mo

Cliquer Save puis sur le bouton "Clear Disk Cache Now" pour initialiser le cache.

05

Filtrage avec SquidGuard

Services→SquidGuard Proxy Filter→General Settings
  1. Cocher Enable → Save

Télécharger les listes de blocage (Blacklists) :

Services→SquidGuard Proxy Filter→Blacklist
ChampValeur
Blacklist URLhttp://dsi.ut-capitole.fr/blacklists/download/blacklists.tar.gz

Cliquer Download et attendre le téléchargement de la base de données.

Créer des règles de filtrage :

Services→SquidGuard Proxy Filter→Common ACL
CatégorieAction recommandée
adult / porn🚫 Deny
malware / phishing🚫 Deny
social_networks🚫 Deny (selon politique)
games🚫 Deny (selon politique)
default✅ Allow

Cliquer Save puis aller dans General Settings → cliquer Apply pour activer les règles.

⚠️

Après chaque modification dans SquidGuard, cliquer sur le bouton "Apply" dans General Settings. Sans cette étape, les règles ne sont pas prises en compte.

06

Logs et vérification

Consulter les logs de Squid pour vérifier le fonctionnement :

Status→System Logs→Packages→Squid

Pour vérifier que le proxy fonctionne depuis un poste client, ouvrir un navigateur et tester l'accès à un site. Puis vérifier dans les logs :

Services→Squid Proxy Server→Real Time

Les logs en temps réel affichent chaque requête avec le format suivant :

log
1710000000.123 456 192.168.2.22 TCP_MISS/200 12345 GET http://example.com/ - DIRECT/93.184.216.34 text/html ↑ ↑ ↑ ↑ ↑ ↑ ↑ ↑ ↑ timestamp délai client IP résultat taille méthode URL serveur type MIME
Code résultatSignification
TCP_HITRéponse servie depuis le cache local ✅
TCP_MISSRequête transmise au serveur distant (pas en cache)
TCP_DENIEDRequête bloquée par SquidGuard 🚫
TCP_TUNNELTrafic HTTPS tunnelisé (non filtré)
✅

Squid est opérationnel en mode proxy transparent sur pfSense. Le trafic HTTP est mis en cache, journalisé et filtré par SquidGuard selon les catégories configurées.
