Chapitre 14 Active Directory GUI

Stratégies GPO

Création et application de stratégies de groupe (GPO) pour la sécurité, la personnalisation du bureau et la restriction des logiciels.

01

Introduction

Les stratégies de groupe (GPO — Group Policy Objects) permettent d'appliquer automatiquement des configurations à des utilisateurs ou des ordinateurs d'une OU. Elles sont gérées via la console GPMC.

Ouvrir la console GPMC :

Gestionnaire de serveur→Outils→Gestion des stratégies de groupe
Type de GPOS'applique àExemples
Configuration ordinateurL'ordinateur au démarragePare-feu, scripts de démarrage, installation logiciels
Configuration utilisateurL'utilisateur à la connexionFond d'écran, restrictions, lecteurs réseau
ℹ️

L'ordre d'application des GPO est : Local → Site → Domaine → OU parente → OU enfant. Une GPO d'OU enfant écrase une GPO de domaine en cas de conflit.

02

Créer et lier une GPO

Dans la console GPMC, faire un clic droit sur l'OU cible :

DOMAINE.PAULR→OU cible→Clic droit→Créer un objet GPO dans ce domaine, et le lier ici
  1. Donner un nom explicite à la GPO (ex: GPO_Securite_Postes)
  2. Cliquer OK — la GPO apparaît liée à l'OU
  3. Faire un clic droit sur la GPO → Modifier pour ouvrir l'éditeur
ℹ️

Nommer les GPO de façon explicite facilite la maintenance : GPO_Securite_Postes, GPO_Bureau_Utilisateurs, GPO_Restriction_Logiciels.

03

GPO de sécurité

Créer une GPO GPO_Securite_Postes liée à l'OU OrdinateursPostes. Dans l'éditeur de stratégie :

Politique de mot de passe :

Configuration ordinateur→Paramètres Windows→Paramètres de sécurité→Stratégies de comptes→Stratégie de mot de passe
ParamètreValeur recommandée
Longueur minimale du mot de passe12 caractères
Le mot de passe doit respecter des exigences de complexitéActivé
Durée de vie maximale du mot de passe90 jours
Conserver l'historique des mots de passe10 mots de passe

Verrouillage de compte :

Configuration ordinateur→Paramètres Windows→Paramètres de sécurité→Stratégies de comptes→Stratégie de verrouillage du compte
ParamètreValeur recommandée
Seuil de verrouillage du compte5 tentatives
Durée de verrouillage du compte30 minutes
Réinitialiser le compteur après30 minutes

Profil DomaineActivé — Connexions entrantes : Bloquer (par défaut)
Profil PrivéActivé — Connexions entrantes : Bloquer (par défaut)
Profil PublicActivé — Connexions entrantes : Bloquer (par défaut)
04

GPO de bureau

Créer une GPO GPO_Bureau_Utilisateurs liée à l'OU Utilisateurs. Dans l'éditeur :

Fond d'écran personnalisé :

Configuration utilisateur→Modèles d'administration→Bureau→Bureau
ParamètreValeur
Papier peint du bureauActivé
Nom du papier peintChemin UNC vers l'image (ex: \\SRV-2025\Partage\fond.jpg)
Style du papier peintÉtirer / Remplir

Désactiver le panneau de configuration :

Configuration utilisateur→Modèles d'administration→Panneau de configuration
ParamètreValeur
Interdire l'accès au panneau de configuration et aux paramètres du PCActivé

Redirection de dossiers (Documents vers le serveur) :

Configuration utilisateur→Paramètres Windows→Redirection de dossiers→Documents→Clic droit→Propriétés
ParamètreValeur
ParamètreDe base — Rediriger le dossier de tout le monde vers le même emplacement
Emplacement du dossier cible\\SRV-2025\Profils\%username%\Documents
05

GPO de restriction logiciel

Créer une GPO GPO_Restriction_Logiciels liée à l'OU Utilisateurs. Dans l'éditeur :

Configuration utilisateur→Paramètres Windows→Paramètres de sécurité→Stratégies de restriction logicielle
  1. Faire un clic droit sur "Stratégies de restriction logicielle" → "Nouvelles stratégies de restriction logicielle"
  2. Double-cliquer sur "Niveau de sécurité par défaut" → le niveau est "Non restreint" par défaut
  3. Pour bloquer des applications spécifiques, aller dans "Règles supplémentaires" → "Nouvelle règle de hachage" ou "Nouvelle règle de chemin"
Type de règleDescriptionExemple
Règle de cheminBloquer/autoriser selon l'emplacement du fichierBloquer C:\Games\*
Règle de hachageIdentifier un exécutable précis par son empreinteBloquer une version précise d'un logiciel
Règle de zone réseauRestreindre selon la zone Internet ExplorerBloquer les exécutables de la zone Internet
⚠️

Tester les GPO de restriction sur un petit groupe d'utilisateurs avant de les déployer à grande échelle. Une restriction mal configurée peut empêcher le démarrage de Windows.

06

Appliquer et tester les GPO

Les GPO s'appliquent automatiquement toutes les 90 minutes sur les postes clients. Pour forcer l'application immédiate depuis le poste client :

Invite de commandes (Admin)→gpupdate /force
CommandeDescription
gpupdate /forceForce le rafraîchissement immédiat de toutes les GPO
gpresult /rAffiche les GPO appliquées à l'utilisateur et l'ordinateur courants
gpresult /h rapport.htmlGénère un rapport HTML détaillé des GPO appliquées

Depuis la console GPMC sur le serveur, utiliser l'outil Résultats de stratégie de groupe :

GPMC→Résultats de stratégie de groupe→Clic droit→Assistant Résultats de stratégie de groupe
✅

Les GPO sont créées et liées aux OU. Elles s'appliqueront automatiquement aux utilisateurs et ordinateurs lors de leur prochaine connexion ou au prochain rafraîchissement.
