Chapitre 16 DHCP GUI

Rôle DHCP

Installation et configuration du rôle DHCP sur Windows Server 2025 : étendue, options, exclusions, réservations et haute disponibilité.

01

Introduction

Le rôle DHCP (Dynamic Host Configuration Protocol) permet d'attribuer automatiquement des adresses IP et des paramètres réseau aux machines du domaine, évitant la configuration manuelle sur chaque poste.

ConceptDescription
Étendue (Scope)Plage d'adresses IP distribuées aux clients
ExclusionAdresses IP réservées hors distribution DHCP (serveurs, imprimantes...)
RéservationAdresse IP fixe attribuée à une machine via son adresse MAC
Options DHCPParamètres supplémentaires transmis aux clients (passerelle, DNS, NTP...)
Bail (Lease)Durée pendant laquelle l'adresse IP est attribuée à un client
⚠️

Si pfSense distribue déjà des adresses IP via son serveur DHCP intégré, penser à le désactiver avant d'activer le serveur DHCP Windows pour éviter les conflits. Aller dans pfSense → Services → DHCP Server → désactiver.

02

Ajout du rôle DHCP

Gestionnaire de serveur→Gérer→Ajouter des rôles et fonctionnalités
  1. Type d'installation → Installation basée sur un rôle → Suivant
  2. Sélectionner le serveur local → Suivant
  3. Cocher "Serveur DHCP" → Ajouter les fonctionnalités → Suivant
  4. Cliquer Suivant jusqu'à la page de confirmation → Installer
  5. Attendre la fin de l'installation puis cliquer Fermer
03

Autorisation dans Active Directory

Un serveur DHCP doit être autorisé dans l'AD pour distribuer des adresses sur un domaine. Une notification apparaît dans le Gestionnaire de serveur après l'installation :

⚑ Notifications→Terminer la configuration DHCP
  1. Cliquer Suivant sur la page d'introduction
  2. Laisser les informations d'identification par défaut (compte Administrateur du domaine)
  3. Cliquer Valider — le serveur DHCP est autorisé dans l'AD
  4. Cliquer Fermer
✅

Le serveur DHCP est maintenant autorisé. Ouvrir la console DHCP via Gestionnaire de serveur → Outils → DHCP.

04

Création d'une étendue

Dans la console DHCP, faire un clic droit sur IPv4 :

IPv4→Clic droit→Nouvelle étendue

L'assistant de création d'étendue s'ouvre. Renseigner les paramètres :

ÉtapeParamètreValeur
Nom de l'étendueNomRéseau LAN 192.168.2.0
Plage d'adresses IPAdresse IP de début192.168.2.100
Plage d'adresses IPAdresse IP de fin192.168.2.200
Plage d'adresses IPMasque de sous-réseau255.255.255.0
Durée du bailDurée8 jours (recommandé pour un réseau stable)

Cliquer Suivant jusqu'à la fin de l'assistant, puis sélectionner "Oui, activer cette étendue maintenant".

05

Options DHCP

Les options DHCP transmettent des paramètres réseau aux clients en plus de l'adresse IP. Les configurer dans l'étendue :

IPv4→Étendue→Options d'étendue→Clic droit→Configurer les options
N° optionNomValeur
003Routeur (passerelle)192.168.2.1 (pfSense)
006Serveurs DNS192.168.2.16 (DC) puis 8.8.8.8
015Nom de domaine DNSDOMAINE.PAULR
044Serveurs WINS/NBNS192.168.2.16 (optionnel)
ℹ️

Les options peuvent être définies à trois niveaux : Serveur (s'applique à toutes les étendues), Étendue (s'applique à une étendue spécifique) ou Réservation (s'applique à un client précis). L'option de niveau inférieur écrase celle de niveau supérieur.

06

Exclusions de plages IP

Les exclusions permettent de réserver des adresses IP dans la plage DHCP pour les équipements à IP statique (serveurs, imprimantes, équipements réseau).

IPv4→Étendue→Plage d'adresses→Clic droit→Ajouter des exclusions et un délai
Plage exclueUsage
192.168.2.1 → 192.168.2.20Équipements réseau et serveurs à IP fixe
192.168.2.250 → 192.168.2.254Équipements d'infrastructure (NAS, imprimantes...)
ℹ️

Dans notre environnement : pfSense (.1), Zabbix (.18), DC (.16), Hyper-V (.11) ont tous des IP fixes. Ils doivent être exclus de la plage DHCP même s'ils ne sont pas configurés en DHCP, pour éviter tout conflit futur.

07

Réservations d'adresses IP

Une réservation garantit qu'une machine reçoit toujours la même adresse IP via DHCP, identifiée par son adresse MAC.

IPv4→Étendue→Réservations→Clic droit→Nouvelle réservation
ChampValeur exemple
Nom de la réservationPoste-Jean-Dupont
Adresse IP192.168.2.150
Adresse MAC00-1A-2B-3C-4D-5E (sans tirets ni deux-points)
DescriptionPC bureau J.Dupont Comptabilité
Types pris en chargeDHCP uniquement

Pour trouver l'adresse MAC d'un poste Windows, exécuter depuis le poste :

cmd
C:\> getmac /v ou C:\> ipconfig /all
08

Haute disponibilité DHCP (Failover)

Le failover DHCP permet d'avoir deux serveurs DHCP qui se partagent la distribution des adresses. Si l'un tombe, l'autre prend le relais automatiquement.

IPv4→Étendue→Clic droit→Configurer le basculement
  1. Cliquer Suivant sur la page d'introduction
  2. Dans "Serveur partenaire", entrer le nom ou l'IP du second serveur DHCP
  3. Cliquer Suivant puis configurer le mode :
ModeDescriptionUsage
Équilibrage de chargeLes deux serveurs distribuent des adresses simultanément (50/50 par défaut)Haute disponibilité + répartition de charge
Serveur de secoursLe serveur secondaire prend le relais uniquement si le primaire est indisponibleFailover simple
✅

Le rôle DHCP est configuré avec étendue, options, exclusions, réservations et haute disponibilité. Les postes du réseau 192.168.2.0/24 obtiennent automatiquement leur configuration réseau.
