Chapitre 05 Machines Virtuelles

Création de VMs

Création pas à pas d'une machine virtuelle Windows et d'une machine virtuelle Debian Linux sur Hyper-V.

01

Prérequis

Avant de créer des VMs, s'assurer d'avoir :

ÉlémentDétail
ISO WindowsImage .iso de Windows Server 2022 ou Windows 10/11
ISO DebianImage .iso de Debian 12 (netinstall ou DVD)
Commutateur virtuelvSwitch-LAN créé (voir chapitre précédent)
Espace disqueMinimum 40 Go libres par VM Windows, 20 Go par VM Linux
RAM disponibleDépend du nombre de VMs à faire tourner simultanément
⚠️

Ne pas démarrer trop de VMs simultanément pour éviter la saturation de la RAM de l'hôte Hyper-V.

02

Création d'une VM Windows

Dans le Gestionnaire Hyper-V, cliquer sur Action → Nouveau → Ordinateur virtuel. Suivre l'assistant avec les paramètres suivants :

ÉtapeParamètreValeur recommandée
Nom et emplacementNomwindows-2025
GénérationGénérationGénération 2 (UEFI, recommandé)
MémoireRAM de démarrage2048 Mo minimum
MémoireMémoire dynamique✅ Activée
RéseauCommutateurvSwitch-LAN
Disque dur virtuelTaille60 Go
InstallationImage ISOSélectionner le fichier .iso Windows

Après création, avant de démarrer la VM, configurer le démarrage sécurisé pour Windows :

powershell
# Activer le démarrage sécurisé avec le template Microsoft Windows PS C:\Windows\system32> Set-VMFirmware -VMName "windows-2025" -SecureBootTemplate MicrosoftWindows

Démarrer la VM et suivre l'installation Windows normalement depuis la console Hyper-V.

ℹ️

Pour accéder à la console de la VM, double-cliquer dessus dans le gestionnaire Hyper-V puis cliquer sur Se connecter.

03

Création d'une VM Debian

Même procédure que Windows, avec quelques différences importantes :

ÉtapeParamètreValeur recommandée
Nom et emplacementNomdebian
GénérationGénérationGénération 1 (compatibilité Linux)
MémoireRAM de démarrage1024 Mo minimum
MémoireMémoire dynamique✅ Activée
RéseauCommutateurvSwitch-LAN
Disque dur virtuelTaille20 Go
InstallationImage ISOSélectionner le fichier .iso Debian
⚠️

Utiliser la Génération 1 pour les VMs Linux sous Hyper-V pour une meilleure compatibilité. La Génération 2 peut causer des problèmes de démarrage avec certaines distributions.

Après l'installation de Debian, installer les Integration Services Linux pour optimiser les performances :

bash
# Installer les outils d'intégration Hyper-V root@debian:~# apt install hyperv-daemons -y # Activer les services root@debian:~# systemctl enable hv-fcopy-daemon hv-kvp-daemon hv-vss-daemon root@debian:~# systemctl start hv-fcopy-daemon hv-kvp-daemon hv-vss-daemon
✅

Avec les Integration Services installés, Hyper-V peut afficher les métriques de la VM (CPU, RAM) directement dans le gestionnaire et les points de contrôle fonctionnent correctement.

04

Ressources & bonnes pratiques

Voici les recommandations pour une allocation optimale des ressources sur l'hôte DESKTOP-HAS36UT :

VMRAM min.RAM max.vCPUDisque
windows-20252 048 Mo4 096 Mo260 Go
windows-101 024 Mo2 048 Mo240 Go
debian512 Mo1 024 Mo120 Go
zabbix (debian)1 024 Mo2 048 Mo220 Go

Bonnes pratiques à respecter :

PratiqueRaison
Activer la mémoire dynamiqueHyper-V ajuste la RAM selon les besoins réels de la VM
Ne pas dépasser 80% de la RAM hôteGarder de la marge pour l'OS hôte et Hyper-V
Utiliser des fichiers VHDXFormat moderne, plus performant que VHD
Stocker les VMs sur un SSDAméliore drastiquement les performances I/O
Faire des points de contrôle avant chaque modifPermet de revenir en arrière en cas de problème
ℹ️

Pour créer un point de contrôle (snapshot), faire un clic droit sur la VM dans le gestionnaire → Point de contrôle. Pour restaurer, sélectionner le point dans le volet inférieur → Appliquer.
