Chapitre 03 Agent Zabbix

Intégration Hyper-V

Installation de l'agent Zabbix sur Hyper-V Server standalone pour superviser les métriques système : CPU, RAM, disque et services Windows.

01

Introduction

Microsoft Hyper-V Server est un hyperviseur standalone gratuit. Contrairement à pfSense, on utilise ici l'agent Zabbix pour Windows qui offre des métriques système bien plus riches que le protocole SNMP.

ParamètreValeur
ÉquipementDESKTOP-HAS36UT
Adresse IP192.168.2.11
MéthodeAgent Zabbix
Port agent10050 (TCP)
02

Téléchargement de l'agent

Depuis le navigateur du serveur Hyper-V, télécharger l'agent sur le site officiel Zabbix :

url
https://www.zabbix.com/download_agents

Sélectionner les options suivantes :

OptionValeur
Version7.0 LTS
OSWindows
Architectureamd64
EncryptionOpenSSL
Format.msi
⚠️

La version de l'agent doit correspondre à la version majeure du serveur Zabbix (ici 7.0).

03

Installation de l'agent

Double-cliquer sur le fichier .msi et renseigner les paramètres dans l'assistant :

ChampValeur
Zabbix server IP192.168.2.18
Agent listen port10050
HostnameHyperV

Vérifier ensuite que le service est bien démarré dans PowerShell (en administrateur) :

powershell
PS C:\Windows\system32> Get-Service "Zabbix Agent 2" Status Name DisplayName ------ ---- ----------- Running Zabbix Agent 2 Zabbix Agent 2
04

Règle pare-feu Windows

Autoriser le port 10050 en entrée pour que le serveur Zabbix puisse interroger l'agent :

powershell
PS C:\Windows\system32> New-NetFirewallRule -DisplayName "Zabbix Agent" ` -Direction Inbound ` -Protocol TCP ` -LocalPort 10050 ` -Action Allow
05

Ajouter l'hôte dans Zabbix

Dans l'interface Zabbix, naviguer vers Configuration → Hôtes → Créer un hôte :

ChampValeur
Nom d'hôteHyperV
GroupesHypervisors
Type d'interfaceAgent Zabbix
Adresse IP192.168.2.11
Port10050
TemplateWindows by Zabbix agent
06

Vérification

Tester la communication depuis le serveur Zabbix :

bash
root@zabbix:~# zabbix_get -s 192.168.2.11 -p 10050 -k agent.ping 1
✅

La réponse "1" confirme que l'agent est accessible. L'hôte Hyper-V apparaît en vert dans Monitoring → Hôtes.
