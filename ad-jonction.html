Chapitre 15 Active Directory GUI

Jonction au domaine

Intégration d'un poste Windows 10/11 au domaine Active Directory DOMAINE.PAULR et vérification de l'application des GPO.

01

Prérequis

Avant de joindre un poste au domaine, vérifier les conditions suivantes :

PrérequisVérification
Le poste est sur le même réseau que le DCping 192.168.2.16 doit répondre
Le DNS du poste pointe vers le DCDNS primaire = 192.168.2.16
Le contrôleur de domaine est démarréServices AD DS actifs sur le serveur
Compte admin du domaine disponibleEx: DOMAINEAdministrateur
⚠️

Le DNS est le point le plus critique. Si le poste ne pointe pas vers le contrôleur de domaine en DNS, la jonction échouera avec une erreur "domaine introuvable".

02

Configuration du DNS

Sur le poste Windows 10/11, configurer le DNS pour qu'il pointe vers le contrôleur de domaine :

Panneau de configuration→Centre Réseau et partage→Modifier les paramètres de la carte→Clic droit sur l'adaptateur→Propriétés→Protocole Internet version 4 (TCP/IPv4)
ParamètreValeur
Adresse IP192.168.2.22 (statique)
Masque de sous-réseau255.255.255.0
Passerelle par défaut192.168.2.1 (pfSense)
Serveur DNS préféré192.168.2.16 (contrôleur de domaine)
Serveur DNS auxiliaire8.8.8.8

Vérifier que le DC est joignable et que le DNS résout le domaine :

Invite de commandes→nslookup DOMAINE.PAULR
✅

Si nslookup retourne l'adresse IP 192.168.2.16 pour DOMAINE.PAULR, le DNS est correctement configuré.

03

Joindre le domaine

Sur le poste Windows 10/11, accéder aux propriétés système :

Paramètres→Système→Informations système→Paramètres avancés du système

Ou via un raccourci clavier :

Win + R→sysdm.cpl→Entrée
  1. Onglet "Nom de l'ordinateur" → cliquer sur "Modifier"
  2. Sélectionner "Domaine" et entrer : DOMAINE.PAULR
  3. Cliquer OK
  4. Entrer les identifiants d'un compte administrateur du domaine :
ChampValeur
Nom d'utilisateurAdministrateur ou DOMAINEAdministrateur
Mot de passeMot de passe de l'administrateur du domaine
  1. Un message "Bienvenue dans le domaine DOMAINE.PAULR" confirme la jonction
  2. Cliquer OK et redémarrer le poste
⚠️

Si le message d'erreur "Le domaine DOMAINE.PAULR est introuvable" apparaît, vérifier que le DNS pointe bien vers le contrôleur de domaine et que le DC est accessible par ping.

04

Vérification dans l'AD

Sur le contrôleur de domaine, vérifier que le poste apparaît dans l'AD :

ADUC→DOMAINE.PAULR→Computers

Le poste Windows 10/11 doit apparaître avec son nom dans le conteneur Computers. Le déplacer vers l'OU dédiée aux postes :

Clic droit sur l'ordinateur→Déplacer→Ordinateurs→Postes
ℹ️

Une fois le poste déplacé dans l'OU OrdinateursPostes, les GPO liées à cette OU s'appliqueront au prochain rafraîchissement (ou après gpupdate /force).

05

Première connexion avec un compte du domaine

Sur le poste redémarré, l'écran de connexion affiche maintenant l'option de connexion au domaine :

  1. Sur l'écran de connexion Windows, cliquer sur "Autre utilisateur"
  2. Entrer le nom d'utilisateur du domaine : j.dupont ou DOMAINEj.dupont
  3. Entrer le mot de passe
  4. Si l'option "L'utilisateur doit changer le mot de passe à la prochaine ouverture de session" était cochée, Windows demande de définir un nouveau mot de passe
ℹ️

La première connexion d'un utilisateur sur un poste crée son profil local (dossier dans C:Usersj.dupont). Si la redirection de dossiers est configurée via GPO, les Documents seront stockés sur le serveur.

06

Test des GPO

Vérifier que les GPO s'appliquent correctement sur le poste client :

Invite de commandes (Admin)→gpupdate /force

Puis vérifier les GPO appliquées :

Invite de commandes (Admin)→gpresult /r
SectionCe qu'on vérifie
COMPUTER SETTINGSGPO appliquées à l'ordinateur (GPO_Securite_Postes...)
USER SETTINGSGPO appliquées à l'utilisateur connecté (GPO_Bureau_Utilisateurs...)

Vérifications visuelles :

GPOVérification visuelle
GPO_Bureau_UtilisateursLe fond d'écran personnalisé est appliqué
GPO_Bureau_UtilisateursLe Panneau de configuration est inaccessible
GPO_Securite_PostesLe pare-feu Windows est actif et configuré
GPO_Restriction_LogicielsLes applications bloquées ne s'exécutent pas
✅

Le poste Windows 10/11 est intégré au domaine DOMAINE.PAULR et reçoit correctement les stratégies GPO. L'infrastructure Active Directory est opérationnelle !
