Chapitre 18 GLPI Linux

Agent GLPI

Installation de l'agent GLPI sur Debian Linux et configuration de la remontée automatique de l'inventaire vers le serveur GLPI.

01

Introduction

GLPI (Gestionnaire Libre de Parc Informatique) est un outil de gestion de parc et de helpdesk open source. L'agent GLPI s'installe sur les machines à inventorier et remonte automatiquement leurs caractéristiques matérielles et logicielles vers le serveur GLPI.

Ce que remonte l'agentExemples
MatérielCPU, RAM, disques, cartes réseau, BIOS
SystèmeOS, version, kernel, hostname
LogicielsListe des paquets installés
RéseauAdresses IP, adresses MAC, interfaces
UtilisateursDernier utilisateur connecté
ℹ️

L'agent GLPI est le successeur de FusionInventory. Si votre serveur GLPI est en version 10+, utiliser l'agent GLPI natif. Pour GLPI 9.x, utiliser le plugin FusionInventory.

02

Prérequis

ÉlémentValeur
Serveur GLPIGLPI 10.x installé et accessible (ex: http://192.168.2.20/glpi)
Machine cibleDebian 12 (dans notre cas la VM 192.168.2.20)
Accès réseauLa machine doit joindre le serveur GLPI en HTTP/HTTPS
DroitsAccès root ou sudo sur la machine cible
⚠️

Vérifier que le serveur GLPI est accessible depuis la machine à inventorier avant d'installer l'agent : curl http://192.168.2.20/glpi

03

Installation sur Debian Linux

Se connecter en SSH ou en console sur la machine Debian cible :

bash
# Mettre à jour le système root@debian:~# apt update && apt upgrade -y # Installer les dépendances root@debian:~# apt install -y wget curl dmidecode perl libio-compress-perl # Télécharger la dernière version de l'agent GLPI root@debian:~# wget https://github.com/glpi-project/glpi-agent/releases/latest/download/glpi-agent_linux-installer.pl # Lancer l'installateur root@debian:~# perl glpi-agent_linux-installer.pl

L'installateur propose plusieurs options. Valider les options par défaut en appuyant sur Entrée ou personnaliser :

QuestionRéponse recommandée
Installation target/usr/local (défaut)
Backend targetsserver (remontée vers serveur GLPI)
Server URLhttp://192.168.2.20/glpi
04

Configuration de l'agent

Le fichier de configuration principal de l'agent se trouve dans /etc/glpi-agent/agent.cfg :

bash
root@debian:~# nano /etc/glpi-agent/agent.cfg

Paramètres clés à vérifier ou modifier :

config
# URL du serveur GLPI server = http://192.168.2.20/glpi # Fréquence d'inventaire (en secondes) — 86400 = 1 fois par jour delaytime = 86400 # Nom de l'agent (optionnel, hostname par défaut) # tag = ServeurDebian # Activer le mode verbeux pour le débogage # debug = 1
05

Remontée vers le serveur GLPI

Démarrer et activer l'agent au démarrage :

bash
# Activer et démarrer le service root@debian:~# systemctl enable glpi-agent root@debian:~# systemctl start glpi-agent # Vérifier le statut du service root@debian:~# systemctl status glpi-agent

Pour forcer une remontée d'inventaire immédiate sans attendre le délai programmé :

bash
# Forcer l'inventaire immédiatement root@debian:~# glpi-agent --force # Voir les logs de l'agent en temps réel root@debian:~# journalctl -u glpi-agent -f
ℹ️

Sur le serveur GLPI, aller dans Administration → Inventaire pour voir les remontées en attente. Les équipements apparaissent dans Parc → Ordinateurs après traitement.

06

Vérification dans GLPI

Se connecter à l'interface web GLPI et vérifier la remontée :

Administration→Inventaire→Imports en attente
  1. La machine Debian doit apparaître dans la liste des imports en attente
  2. Cliquer sur Traiter ou Tout traiter
  3. La machine est ensuite visible dans Parc → Ordinateurs
Onglet dans GLPIInformations disponibles
ComposantsCPU, RAM, disques détectés automatiquement
LogicielsListe complète des paquets Debian installés
RéseauInterfaces réseau et adresses IP/MAC
VolumesPartitions et espace disque disponible
✅

L'agent GLPI est installé et configuré. L'inventaire de la machine Debian remonte automatiquement vers le serveur GLPI toutes les 24 heures.
