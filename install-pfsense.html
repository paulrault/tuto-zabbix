Chapitre 10 pfSense Installation

pfSense 2.7

Installation et configuration initiale de pfSense 2.7.x comme pare-feu/routeur dans une VM Hyper-V avec deux interfaces réseau (WAN et LAN).

01

Prérequis

ÉlémentValeur
ISO pfSense CETéléchargeable sur netgate.com/downloads
RAM allouée1 024 Mo minimum
Disque virtuel10 Go (VHDX)
Génération VMGénération 1
Interfaces réseau2 cartes réseau : une WAN, une LAN
⚠️

pfSense nécessite deux interfaces réseau dans la VM : une connectée au commutateur externe (WAN) et une connectée au commutateur interne ou LAN (pour le réseau local des VMs).

Dans Hyper-V, ajouter une seconde carte réseau à la VM pfSense avant de démarrer :

  1. Clic droit sur la VM → Paramètres
  2. Ajouter un matériel → Carte réseau héritée
  3. Connecter la première carte au commutateur WAN (externe)
  4. Connecter la seconde carte au commutateur LAN (interne ou externe selon votre config)
02

Démarrage sur l'ISO

Au démarrage, l'écran de boot pfSense (FreeBSD) apparaît. Laisser le compte à rebours se terminer ou appuyer sur Entrée pour démarrer immédiatement.

Le système démarre sur un live environnement FreeBSD et lance automatiquement l'installateur pfSense.

03

Installation sur le disque

  1. Sélectionner "Install" sur l'écran d'accueil de l'installateur
  2. Keymap → Sélectionner "French ISO-8859-1" ou continuer avec le clavier par défaut
  3. Partitionnement → Sélectionner "Auto (ZFS)" (recommandé) ou "Auto (UFS)"
  4. Pour ZFS → "Proceed with Installation" → sélectionner le disque virtuel → "Stripe"
  5. Confirmer la destruction des données → "YES"

L'installation copie les fichiers système (2-3 minutes). À la fin :

  1. Sélectionner "No" pour ne pas ouvrir de shell de configuration
  2. Sélectionner "Reboot"
  3. Retirer l'ISO depuis les paramètres Hyper-V avant le redémarrage
04

Configuration des interfaces réseau

Au premier démarrage, pfSense détecte les interfaces et demande de les assigner. Dans la console :

  1. "Should VLANs be set up now?" → n
  2. "Enter the WAN interface name" → ex: hn0 (première carte réseau Hyper-V)
  3. "Enter the LAN interface name" → ex: hn1 (seconde carte réseau)
  4. Confirmer l'assignation → y
InterfaceRôleIP par défaut
hn0 → WANConnexion vers l'extérieur / réseau hôteDHCP (obtenue automatiquement)
hn1 → LANRéseau local des VMs192.168.1.1/24
ℹ️

Dans notre environnement, pfSense a l'IP LAN 192.168.2.1. L'IP par défaut 192.168.1.1 peut être modifiée depuis la console ou l'interface web.

05

Accès à l'interface web

Depuis une VM connectée au réseau LAN de pfSense, ouvrir un navigateur et accéder à :

url
https://192.168.1.1
ChampValeur par défaut
Identifiantadmin
Mot de passepfsense
⚠️

Changer immédiatement le mot de passe par défaut lors de la première connexion. L'assistant de configuration se lance automatiquement.

L'assistant de configuration initiale (Setup Wizard) guide à travers :

  1. Nom d'hôte et domaine
  2. Serveur DNS
  3. Configuration du WAN
  4. Configuration du LAN (IP et masque)
  5. Changement du mot de passe admin
06

Configuration réseau LAN

Pour changer l'IP LAN de pfSense vers 192.168.2.1, aller dans Interfaces → LAN :

ParamètreValeur
IPv4 Configuration TypeStatic IPv4
IPv4 Address192.168.2.1
Subnet Mask/24 (255.255.255.0)

Configurer le serveur DHCP pour le réseau LAN dans Services → DHCP Server → LAN :

ParamètreValeur
Enable✅ Coché
Range From192.168.2.100
Range To192.168.2.200
DNS Server192.168.2.1
07

Sécurité de base

Dans System → Advanced → Admin Access :

ParamètreRecommandation
ProtocolHTTPS uniquement
TCP Port8443 (port non standard)
Anti-lockout Rule✅ Activé
SSHDésactiver si non nécessaire

Dans System → Update, vérifier et installer les dernières mises à jour de pfSense.

✅

pfSense 2.7 est installé et configuré. Le pare-feu est opérationnel. Vous pouvez maintenant activer SNMP pour le superviser avec Zabbix (voir chapitre Intégration pfSense).
