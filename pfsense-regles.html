Chapitre 17 pfSense GUI

Règles pfSense

Création de règles de pare-feu, configuration du NAT et redirection de ports dans pfSense 2.7 via l'interface web.

01

Introduction

pfSense filtre le trafic réseau à l'aide de règles de pare-feu appliquées sur chaque interface. Les règles sont évaluées de haut en bas — la première règle correspondante est appliquée.

ConceptDescription
Règle de pare-feuAutorise ou bloque le trafic selon des critères (IP source, destination, port, protocole)
AliasGroupe nommé d'adresses IP ou de ports pour simplifier les règles
NAT OutboundTraduction d'adresse pour le trafic sortant vers Internet (masquage d'IP)
Port ForwardRedirige le trafic entrant sur un port WAN vers un serveur interne (LAN)
Floating RulesRègles globales s'appliquant à toutes les interfaces
ℹ️

pfSense applique une règle implicite "Tout bloquer" à la fin de chaque liste. Tout trafic non explicitement autorisé est bloqué. Sur l'interface LAN, une règle "Allow LAN to any" est créée par défaut.

02

Créer des alias

Les alias simplifient la gestion des règles en regroupant des IPs ou des ports sous un nom. Les modifier en un seul endroit met à jour toutes les règles qui les utilisent.

Firewall→Aliases→Add

Alias d'adresses IP (serveurs) :

NomTypeContenu
SERVEURS_INTERNESHost(s)192.168.2.11, 192.168.2.16, 192.168.2.18
DNS_EXTERNESHost(s)8.8.8.8, 1.1.1.1

Alias de ports :

NomTypePorts
PORTS_WEBPort(s)80, 443
PORTS_ADMINPort(s)22, 3389, 8443
PORTS_ZABBIXPort(s)10050, 10051, 161

Cliquer Save puis Apply Changes après chaque alias.

03

Règles de pare-feu

Les règles LAN contrôlent ce que les machines du réseau local peuvent faire. Naviguer vers :

Firewall→Rules→LAN

Règles LAN recommandées :

ActionProtocoleSourceDestinationPortDescription
✅ PassTCP/UDPLAN netanyPORTS_WEBAutoriser navigation web
✅ PassUDPLAN netDNS_EXTERNES53Autoriser DNS externe
✅ PassTCP/UDPLAN netSERVEURS_INTERNESPORTS_ZABBIXAutoriser supervision Zabbix
✅ PassTCP192.168.2.18LAN net10050Zabbix interroge les agents
🚫 BlockTCPLAN netanyPORTS_ADMINBloquer accès admin depuis LAN

Pour ajouter une règle, cliquer sur Add ↑ (en haut de liste, priorité haute) :

  1. Action → Pass / Block / Reject
  2. Interface → LAN (ou WAN selon le sens du trafic)
  3. Address Family → IPv4
  4. Protocol → TCP, UDP, TCP/UDP, ICMP...
  5. Source → réseau source (ex: LAN net ou IP précise)
  6. Destination → réseau ou alias de destination
  7. Destination Port Range → port ou alias de ports
  8. Description → toujours renseigner pour documenter la règle
  9. Cliquer Save puis Apply Changes
⚠️

Les règles sont évaluées de haut en bas. Placer les règles de blocage avant les règles d'autorisation générales. Une règle "Block all" placée en bas n'aura aucun effet si une règle "Pass any" la précède.

04

NAT Outbound

Le NAT Outbound (masquage) permet aux machines du LAN d'accéder à Internet en utilisant l'IP WAN de pfSense. Par défaut, pfSense est en mode Automatic Outbound NAT.

Firewall→NAT→Outbound
ModeDescription
Automatic Outbound NATpfSense crée automatiquement les règles NAT — recommandé pour la plupart des cas
Hybrid Outbound NATRègles automatiques + règles manuelles personnalisées
Manual Outbound NATContrôle total — toutes les règles sont définies manuellement

Pour une configuration standard, laisser le mode Automatic. Vérifier que des règles NAT sont bien générées pour le réseau 192.168.2.0/24 vers l'interface WAN.

05

Redirection de ports (Port Forward)

La redirection de ports permet d'exposer un service interne vers l'extérieur. Exemple : exposer l'interface Zabbix (192.168.2.18:80) depuis le WAN.

Firewall→NAT→Port Forward→Add
ParamètreValeur exemple
InterfaceWAN
ProtocolTCP
DestinationWAN address
Destination Port Range8080 (port externe)
Redirect Target IP192.168.2.18 (serveur Zabbix)
Redirect Target Port80 (port interne)
DescriptionAccès Zabbix depuis WAN
Filter rule associationAdd associated filter rule (crée automatiquement la règle firewall)
✅

Cliquer Save puis Apply Changes. La règle de redirection de port est active. Tester depuis l'extérieur en accédant à http://IP_WAN:8080.

06

Logs et vérification

pfSense journalise le trafic bloqué et autorisé. Pour consulter les logs :

Status→System Logs→Firewall
ColonneDescription
TimeHorodatage de l'événement
InterfaceInterface concernée (LAN, WAN...)
ActionPass (✅) ou Block (🚫)
SourceIP et port source
DestinationIP et port de destination
ProtoProtocole (TCP, UDP, ICMP...)

Pour activer la journalisation sur une règle spécifique, cocher l'option "Log packets that are handled by this rule" dans les propriétés de la règle.

ℹ️

L'outil Packet Capture (Diagnostics → Packet Capture) permet de capturer le trafic en temps réel sur une interface, très utile pour déboguer les règles.
