Chapitre 04 Hyper-V

Prise en main Hyper-V

Présentation, installation du rôle Hyper-V, découverte de l'interface du gestionnaire et configuration des commutateurs virtuels.

01

Présentation d'Hyper-V

Hyper-V est la solution de virtualisation native de Microsoft, disponible dans Windows Server et Windows 10/11 Pro. Il permet de créer et gérer des machines virtuelles (VMs) directement sur un hôte Windows.

Il existe deux variantes principales :

VarianteDescriptionUsage
Rôle Hyper-VIntégré à Windows Server ou Windows 10/11 ProServeur polyvalent
Hyper-V ServerOS standalone gratuit, sans interface graphiqueHyperviseur dédié
ℹ️

Dans ce tutoriel, l'hôte Hyper-V est la machine DESKTOP-HAS36UT avec l'adresse IP 192.168.2.11.

Les composants clés d'Hyper-V :

ComposantRôle
HyperviseurCouche logicielle qui gère les VMs et l'accès au matériel
Commutateur virtuelÉquipement réseau virtuel qui connecte les VMs entre elles et au réseau physique
Disque dur virtuel (VHD/VHDX)Fichier simulant un disque dur pour chaque VM
Point de contrôleSnapshot de l'état d'une VM à un instant T (équivalent snapshot VMware)
Integration ServicesPilotes et services installés dans la VM pour optimiser les performances
02

Installation du rôle Hyper-V

Sur Windows 10/11 Pro ou Windows Server, activer le rôle Hyper-V via PowerShell en administrateur :

powershell
# Activer le rôle Hyper-V (Windows 10/11 Pro) PS C:\Windows\system32> Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Hyper-V -All # Sur Windows Server PS C:\Windows\system32> Install-WindowsFeature -Name Hyper-V -IncludeManagementTools -Restart
⚠️

Un redémarrage est nécessaire après l'activation. Assurez-vous que la virtualisation est activée dans le BIOS/UEFI (option Intel VT-x ou AMD-V).

Vérifier que le rôle est bien installé après redémarrage :

powershell
PS C:\Windows\system32> Get-WindowsOptionalFeature -Online -FeatureName Microsoft-Hyper-V FeatureName : Microsoft-Hyper-V State : Enabled
03

Interface du Gestionnaire Hyper-V

Le Gestionnaire Hyper-V s'ouvre depuis le menu Démarrer → Gestionnaire Hyper-V. L'interface est divisée en trois zones principales :

ZoneDescription
Volet gaucheArborescence des hôtes Hyper-V connectés
Volet centralListe des VMs avec leur état, CPU, RAM et uptime
Volet droit (Actions)Actions disponibles sur l'hôte ou la VM sélectionnée
Volet inférieurPoints de contrôle et détails de la VM sélectionnée

Les états possibles d'une VM :

ÉtatSignification
ExécutionVM démarrée et active
DésactivéVM éteinte
SuspenduVM mise en pause (état sauvegardé en RAM)
EnregistréVM hibernée (état sauvegardé sur disque)
ℹ️

Double-cliquer sur une VM dans le gestionnaire ouvre sa console vidéo, permettant d'interagir directement avec elle comme si vous étiez devant l'écran physique.

04

Configuration des commutateurs virtuels

Les commutateurs virtuels (virtual switches) permettent de définir comment les VMs accèdent au réseau. Il en existe trois types :

TypeConnectivitéUsage typique
ExterneVM ↔ Réseau physique ↔ InternetAccès réseau normal pour les VMs
InterneVM ↔ Hôte uniquementCommunication VMs ↔ hôte sans accès externe
PrivéVM ↔ VM uniquementRéseau isolé entre VMs

Pour créer un commutateur virtuel externe, ouvrir le Gestionnaire de commutateur virtuel depuis le volet Actions :

ParamètreValeur
TypeExterne
NomvSwitch-LAN
Carte réseau associéeCarte réseau physique connectée au réseau local
Autoriser le SE de gestion✅ Coché (pour que l'hôte garde l'accès réseau)
✅

Une fois le commutateur externe créé, les VMs qui lui sont connectées obtiennent une adresse IP du réseau local (via DHCP ou configuration statique) et peuvent communiquer avec les autres machines du réseau.
